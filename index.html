Giles is a re-implementation of the sMAP Archiver in Go. It is part of an effort to modernize (and multi-thread) the sMAP architecture to help it scale to larger deployments.

Installation

Giles is designed to be easy to install: 
$ go get github.com/gtfierro/giles/src/archiver
$ archiver -h
Usage of archiver:
  -cpuprofile="": write cpu profile to file
  -memprofile="": write memory profile to this file
  -mongoip="localhost": MongoDB IP address
  -mongoport=27017: MongoDB Port
  -port=8079: archiver service port
  -rdbip="localhost": ReadingDB IP address
  -rdbport=4242: ReadingDB Port

You will need both ReadingDB and MongoDB running. Point Giles to their ports using the command line options. Giles assumes by default that ReadingDB and MongoDB are running on localhost on ports 4242 and 27017 respectively (these are the default ports for those services)

Status

Giles is still in development, so not all features are implemented

Features

  • Compatibility with ReadingDB (Adaptive Branch)
  • Metadata store over MongoDB (instead of the usual Postgres HSTORE)
  • Support for basic metadata queries
  • Support for data queries

Eventually, I want to implement the full query language described here

Known Issues

  • DELETE queries do not work
  • The DISTINCT keyword does not work
  • Nested quotes are not supported
  • No operators on data queries
  • No nested booleans in WHERE clauses, e.g. "X and Y" is okay, but "X and (Y or Z)" is not
  • "~" and LIKE operators for WHERE clauses not supported

Note: "not supported" and "does not work" are temporary states -- I plan on implementing all these features
